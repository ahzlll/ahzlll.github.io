前言

作为后端开发新手，最近开始学习代码审查（其实就是看别人的代码，学习经验…）。在这个过程中，发现了一些共性问题，也学到了很多。想记录一下自己的思考，也希望能帮助到和我一样的初学者。

代码审查的原则

  1. 对事不对人：指出问题，不要攻击人
  2. 给出建议：不仅要指出问题，还要给出改进建议
  3. 关注重点：优先关注逻辑错误、安全问题、性能问题
  4. 及时反馈：不要等到最后才审查，要持续审查

如何写好代码

  1. 单一职责：每个类、每个方法只做一件事
  2. 命名清晰：变量名、方法名要能表达意图
  3. 异常处理：不要吞掉异常，要适当处理和记录
  4. 代码复用：提取公共逻辑，避免重复
  5. 测试覆盖：重要逻辑要有单元测试

总结

代码审查是提高代码质量的重要手段。通过审查，不仅能发现bug，还能学习到更好的编码方式。

写代码时，要时刻想着：

  • 这段代码别人能看懂吗？
  • 如果出问题，能快速定位吗？
  • 这段代码容易维护吗？

思考：作为初学者，刚开始写代码时，我只关注功能能不能实现。通过代码审查，我意识到好的代码不仅要能运行，还要易读、易维护、易扩展。

虽然我现在还写不出完美的代码，但至少知道了努力的方向吧：

  • 写代码时要多思考，不能只关注功能
  • 代码审查是学习的好机会，要多看多学
  • 要时刻想着：这段代码别人能看懂吗？容易维护吗？

路还很长，继续努力！虽然现在代码写得还不够好，但至少知道努力的方向了 😊
