处理流程：

  1. 创建 ServiceA：

    • 实例化 ServiceA（调用构造函数）
    • 将 ServiceA 的 ObjectFactory 放入三级缓存
    • 开始属性注入，发现需要 ServiceB

  2. 创建 ServiceB：

    • 实例化 ServiceB（调用构造函数）
    • 将 ServiceB 的 ObjectFactory 放入三级缓存
    • 开始属性注入，发现需要 ServiceA
    • 从三级缓存获取 ServiceA 的 ObjectFactory，创建 ServiceA 的早期引用
    • 完成 ServiceB 的属性注入和初始化

  3. 完成 ServiceA：

    • 从二级缓存获取 ServiceA 的早期引用
    • 完成 ServiceA 的属性注入和初始化

总结

Spring IoC 容器的工作流程：

  1. 容器初始化：创建 BeanFactory，加载配置
  2. BeanDefinition 注册：扫描组件，注册 Bean 元数据
  3. Bean 实例化：
    • 实例化（调用构造函数）
    • 属性注入（依赖注入）
    • 初始化（调用初始化方法）
  4. Bean 使用：从容器获取 Bean 实例

关键点：

  • BeanDefinition：存储 Bean 的元数据
  • BeanPostProcessor：Bean 的后置处理器，可以修改 Bean
  • 三级缓存：解决循环依赖问题
  • 延迟初始化：单例 Bean 在容器启动时创建，原型 Bean 在获取时创建

思考：理解 Spring IoC 容器的源码，不仅能帮助我们更好地使用 Spring，还能学习到优秀的设计模式和架构思想。控制反转、依赖注入、工厂模式、策略模式等，都在 Spring 源码中得到了很好的体现。

看源码确实很累（我看了好几天才勉强理解 😭），但收获很大。但毕竟还在学习中，难免会有理解不到位的地方。如果文章有理解错误的地方，欢迎指正！
